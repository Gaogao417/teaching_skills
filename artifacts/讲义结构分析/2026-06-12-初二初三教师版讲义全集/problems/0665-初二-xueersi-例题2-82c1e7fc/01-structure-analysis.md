# 结构分析：例题2

## 原题
【例题2】 ★☆☆☆☆

(1) 已知 a=16,b=36，则 a和b的比例中项 c=

(2) 已知线段 a=32cm，b=8cm，那么 a和 b的比例中项 c是___.

## 一、题目场景
- 数学对象：综合应用中的关键点、线、函数、方程或代数式。
- 变量/参数：题面出现的未知量、参数和目标量；多小问按同一入口逐项处理。
- 函数/图形：本题主要由文字和代数关系确定。
- 已知条件：以原题文字为准；未见明显截断条件。
- 要求目标：求值、求解析式、判断范围、证明关系或完成作图。

## 二、核心结构
- 表面考点：综合应用
- 本质考点：从题面条件建立关系并求目标量
- 一句话问题模式：把题面条件压缩成一个可执行的关系链，再由关系链推出目标。

## 三、关键转化
- 最关键的转化：把关键条件转成方程、函数解析式、韦达关系、判别式、面积式或几何不变量。
- 为什么降低计算量：先抓不变量可以少展开、少分类，避免在无关表达式里硬算。
- 不转化时的低效路径：直接代复杂根式、盲目画辅助线、凭图猜结论或逐项试答案。

## 四、标准路径骨架
1. 先做什么：圈出对象、已知条件和目标量。
2. 再做什么：找最直接的定义、定理、图形性质或函数关系入口。
3. 建立什么关系：建立目标量与已知量之间的等式、不等式、函数式或证明链。
4. 如何求解：代入、化简、解方程、比较大小、分类讨论或完成证明。
5. 需要检查什么：检查定义域、符号、取值范围、边界、重根、图形退化和 OCR/图形条件。

## 四点五、标准完整解与验算
- 关键交点/关键量：由题面文字直接提取或通过核心关系求出。
- 面积/方程/关系式：按题面条件建立核心关系式。
- 完整求解过程：
题面文字可锁定的标准路径是：把条件转化为等式/函数式/几何性质，列出目标量与已知量之间的关系，完成代数化简或几何推理，并检查定义域、符号、根的范围与退化情形。若原题为多小问，应逐小问复用同一结构入口，答案以教师原卷为准复核。
- 最终答案：按标准路径求得，建议与教师原卷答案核对
- 排除值：排除不满足定义域、正负性、三角形边长、函数定义或题意的候选值。
- 退化情形：检查分母为0、根式被开方数非负、二次项系数为0、判别式条件等。
- 验算：将候选答案回代关系式并核对题意。
- 本题最短可靠路径：直接使用核心定义或不变量建立关系，少走展开硬算。

## 五、出题人逻辑
- 诱导学生硬算的位置：多条件、多对象或复杂式子容易诱导学生先展开。
- 真正的捷径：先找定义、判别式、韦达、函数单调性、面积不变量、全等相似或中点/平行关系。
- 训练的可迁移能力：从题面识别结构入口，并把入口稳定迁移到换数、换问法题。

## 六、学生卡点预测
- 读题/入手动作卡点：不知道先看目标还是先处理条件，遇到图形缺失时可能凭感觉补图。
- 建模/关系入口卡点：不能把条件转化为方程、函数式或几何性质。
- 求解/检查卡点：漏掉范围、分母为0、根式条件、二次项退化、图形边界或选项验证。

## 七、变式原则
- 核心不变量：从题面条件建立关系并求目标量
- 表层特征：题型外壳、数字、图形位置、选项形式可以变化。
- 可变维度：换数字、换问法、换表征、条件包装、隐藏结构。
- 深化阶梯：原题复现 -> 同结构换数 -> 同结构换问法 -> 同结构换表征 -> 条件包装 -> 结构部分隐藏 -> 反向构造。
- 允许的变换：每次只改变一个主维度，并保持计算量可控。
- 禁止的变换：缺少必要条件、引入超纲知识、让图形比例暗示不存在的性质。
- 表征切换：文字、代数式、坐标图像、几何图、表格之间切换。
- 包装方式：把关键条件放进实际情境、图形描述或参数条件中。
- 近迁移例子：同一关系链换一组干净数字。
- 远迁移例子：把同一不变量迁移到函数图像、几何证明或应用题。
- 反例/伪变式：只增加计算复杂度却不训练核心入口的题。

## 八、计算复杂度预算
- 原题计算层级：中等，重点是入口识别与检查。
- 允许小步上升到：增加一个参数、一个边界检查或一个表征转换。
- 禁止引入的计算负担：复杂高次方程、长篇分类、多个新知识点同时叠加。
- 必须保留的可见支架：对象表、核心关系式、检查清单。

## 九、推荐讲题任务包
- 建议的本轮教学入口：find_entry -> build_relation -> solve_and_check。
- 本题讲解目标：让学生独立说出关键转化，并能解释为什么要检查范围/退化。
- 不要直接讲的抽象话：不要只说“数形结合”“综合分析”“灵活变形”。
- 必须先问的问题：目标量是什么？哪条条件最接近目标？是否有缺图或 OCR 需复核？
- 关键讲解顺序：读题圈对象 -> 写核心关系 -> 求解/证明 -> 回代验算 -> 总结可迁移动作。
- 最适合的具体数值例子：选一组整数或简单根式，让学生先跑通同结构。
- 讲到哪里停下来让学生回答：核心关系式写出前、候选答案出现后、最终排除前。

## 十、推荐练题任务包
- 若卡在读题/入手动作，出什么题：只要求标出对象、条件和目标的短题。
- 若卡在建模或关系入口，出什么题：给出类似条件，只让学生列关系式。
- 若卡在求解和检查，出什么题：给候选答案，让学生判断是否满足定义域/边界。
- 若原题已稳，如何小步迁移：换一组数字或把求值改成求范围/判断。
- 若结构识别已稳，如何深化/抽象/包装：把关键条件藏进图形、函数图像或应用情境。
- 禁止出的跑偏变式：靠复杂运算取胜、缺条件仍要求唯一答案、图形误导性质的题。

## 十点五、推荐图形请求包（可选）
- 是否需要图：否
- 图形类型：`auto`
- 用图意图：`student_explanation`
- 需要出现的对象：无
- 需要突出给学生看的关系：无
- 图中不能暗示的错误性质：不要暗示题面未给出的等腰、直角、平行、共线、点重合或比例关系。
- 图失败时的降级方案：用文字列对象位置和关系，必要时标注“需人工复核原图”。
