# 结构分析：函数求点坐标条件专题 08 · 平行四边形与特殊平行四边形

## 原题
本专题研究函数求点坐标题中，题目用“平行四边形、矩形、菱形、正方形”等四边形条件限制待求点坐标。典型题面为：已知若干定点，另有动点在一次函数轨迹上，要求四点按固定顺序或无固定顺序构成平行四边形及其特殊类型，求未知点坐标。

## 一、题目场景
- 数学对象：坐标平面中的点、一次函数轨迹、平行四边形和特殊平行四边形。
- 变量/参数：待求点 $C(t,kt+b)$、第四点 $D$ 或候选点对 $(C,D)$。
- 函数/图形：一次函数图像、四边形顶点顺序。
- 已知条件：定点坐标、动点轨迹、四边形类型与顺序说明。
- 要求目标：由四边形性质建立坐标方程，求候选点或候选点对。

## 二、核心结构
### 2.1 表层信息
- 表面考点：对角线中点、平行四边形第四点、矩形直角、菱形等边。
- 题型功能：`existence_or_parameter`
- 是否值得完整 structural analysis：是；核心是先判断题目怎样给“坐标条件”，再决定固定顺序或分类。
- 一句话问题模式：四边形条件先转为中点、向量、垂直或等长方程，再和函数轨迹联立。

### 2.2 结构表达
#### 判别条件表（概念辨析题用；不适用则写“无”）
- 必要条件：平行四边形是特殊四边形的底层条件；矩形还要直角；菱形还要邻边相等；正方形两者都要。
- 充分条件：固定顺序 $ABCD$ 中 $A+C=B+D$ 可确定平行四边形第四点。
- 常见干扰项：把“平行四边形 $ABCD$”和“四点构成平行四边形”混为一类。
- 最短检查动作：先读题面顺序，再写对角点或分类表。

#### 情景量表（应用题用；不适用则写“无”）
| 量 | 类型/单位 | 题设关系 | 未知/已知 |
|---|---|---|---|
| $A,B,C$ | Point2D | 三个顶点或部分动点 | 已知/待设 |
| $D$ | Point2D | 第四点 | 待求 |
| $L$ | LinearFunction | 动点所在轨迹 | 已知 |
| 四边形条件 | QuadrilateralConstraint | 固定顺序或无序构成 | 已知 |

#### 命题网络（所有题型都写；简单题写简版）
- P1（题设）：已知部分顶点坐标和动点轨迹。
- P2（题设）：题面给出四边形类型与顶点顺序。
- P3（定义）：平行四边形固定顺序满足对角线中点相同，即 $A+C=B+D$。
- P4（定义）：无序“构成”题需要讨论三组对角配对。
- P5（定义）：矩形可由平行四边形加相邻边垂直筛选。
- P6（定义）：菱形可由平行四边形加邻边相等或对角线垂直筛选。
- P7（目标）：得到合法候选点或候选点对，并排除退化。
- R1：P1 + P2 + P3 -> P7，方法：固定顺序第四点公式。
- R2：P1 + P2 + P4 -> P7，方法：三种对角分类生成候选点。
- R3：P7 + P5/P6 -> 特殊四边形候选，方法：垂直或等长筛选。
- 目标：从四边形文字条件推出坐标方程，而不是凭图形猜点。

### 2.3 解题主链
```text
读顺序 -> 设动点 -> 平行四边形中点/向量条件 -> 特殊条件筛选 -> 候选点验算
```

### 2.4 模型标签
- model_id：`coordinate_parallelogram_special_existence`
- model_name：一次函数轨迹上的平行四边形及特殊平行四边形存在性
- configuration：已知定点和动点轨迹，要求点坐标或候选点对。
- 可迁移方向：固定顺序第四点、无序候选点、矩形/菱形/正方形筛选。
- 非同构边界：没有明确顺序却只按一种公式求，或只满足平行就宣称特殊四边形。

## 二点五、知识点/模型锚点
- 建议讲义标题：模型八：平行四边形条件怎样给点坐标
- 知识点/模型名称：对角线中点与特殊条件筛选
- 核心公式/定理：固定顺序 $ABCD$ 中 $D=A+C-B$；无序构成有 $A+B-C,A+C-B,B+C-A$ 三类；矩形看垂直，菱形看等长。
- 使用条件：顶点顺序清楚，点不重合，四边形不退化。
- 入口信号：“四边形 $ABCD$ 是……”表示顺序固定；“四点构成……”表示顺序未固定。
- 易混边界：正方形必须同时满足矩形和菱形条件。
- 本题如何体现：四边形条件把待求点坐标变成中点、垂直或距离方程。
- 可作为例题的结构层级：固定第四点 -> 两动点中点联立 -> 特殊平行四边形筛选。

## 三、关键转化
- 最关键的转化：四边形文字条件 $\rightarrow$ 对角线中点关系/向量关系/垂直等长方程。
- 为什么降低计算量：先用平行四边形确定候选，再做特殊条件筛选，避免一开始联立过多未知量。
- 不转化时的低效路径：凭图形猜 $D$，或把矩形/菱形各自从头建复杂方程。

## 四、标准路径骨架
1. 先做什么：读清题目是固定顺序还是无序构成。
2. 再做什么：把所有动点坐标写出来。
3. 建立什么关系：平行四边形用对角线中点；矩形加垂直；菱形加等长。
4. 如何求解：先求候选点或候选点对，再逐类筛选。
5. 需要检查什么：点重合、三点共线、顺序错误、特殊条件是否充分。

## 四点五、标准完整解与验算
- 关键交点/关键量：对角线中点、向量、边长平方、斜率或数量积。
- 面积/方程/关系式：$A+C=B+D$；$(\vec{AB})\cdot(\vec{BC})=0$；$AB^2=BC^2$。
- 完整求解过程：判顺序、设点、列中点方程、求候选、用特殊条件筛选。
- 最终答案：固定顺序通常单个点；无序或特殊筛选可能是候选集。
- 排除值：点重合、零面积、候选不在轨迹上。
- 退化情形：三点共线或第四点与已知点重合。
- 验算：回代中点关系、垂直关系和边长关系。
- 本题最短可靠路径：平行四边形先行，特殊条件后筛。

## 五、出题人逻辑
- 诱导学生硬算的位置：题面出现特殊四边形时直接列很多距离方程。
- 真正的捷径：先把“平行四边形”作为底层关系，把候选点压缩出来。
- 训练的可迁移能力：读懂四边形条件给出的坐标关系和分类边界。

## 六、学生卡点预测
- 读题/入手动作卡点：分不清固定顺序和无序构成。
- 建模/关系入口卡点：不会从中点相同反写未知点坐标。
- 求解/检查卡点：特殊四边形只验证一个条件，漏掉退化或漏掉正方形交集。

## 七、变式原则
- 核心不变量：四边形性质先转成坐标关系，再与轨迹联立。
- 表层特征：定点个数、动点轨迹、固定顺序/无序、特殊类型。
- 可变维度：第四点求法、双动点、候选点筛选、特殊类型。
- 深化阶梯：固定第四点 -> 无序三类 -> 动点在函数上 -> 矩形/菱形筛选 -> 正方形交集。
- 允许的变换：换定点、换一次函数、换特殊条件。
- 禁止的变换：没有顺序说明却只给一种答案；把非平行四边形直接筛成矩形/菱形。
- 表征切换：坐标公式、向量走法、中点表格、函数轨迹。
- 包装方式：从一组候选点中筛特殊四边形，或让动点在两条直线上。
- 近迁移例子：给 $A,B,C$，固定顺序求 $D$。
- 远迁移例子：先由面积条件得到 $C$ 候选，再筛出能构成矩形的 $D$。
- 反例/伪变式：只要求用公式 $D=A+C-B$ 重复换数，不训练条件识别。

## 八、计算复杂度预算
- 原题计算层级：坐标加减、一元一次方程、少量边长平方。
- 允许小步上升到：一个双动点中点联立或三候选点筛选。
- 禁止引入的计算负担：复杂二次方程组、多条曲线轨迹。
- 必须保留的可见支架：顶点顺序、函数轨迹、特殊条件至少要明确给出。

## 九、推荐讲题任务包
- 建议的本轮教学入口：先让学生划出“$ABCD$ 是”还是“四点构成”。
- 本题讲解目标：建立“先平行四边形，后特殊筛选”的顺序。
- 不要直接讲的抽象话：不要只说“利用性质”。
- 必须先问的问题：哪两个点是对角点？有没有固定顺序？
- 关键讲解顺序：读顺序 -> 写点 -> 中点关系 -> 求候选 -> 特殊筛选。
- 最适合的具体数值例子：$A(1,2),B(5,2),C(7,5)$，固定顺序求 $D$。
- 讲到哪里停下来让学生回答：写出 $D=A+C-B$ 后，让学生自己代坐标。

## 十、推荐练题任务包
- 若卡在读题/入手动作，出什么题：固定顺序和无序构成的对比题。
- 若卡在建模或关系入口，出什么题：两动点在直线上，用中点相同联立。
- 若卡在求解和检查，出什么题：候选点筛矩形、菱形、正方形。
- 若原题已稳，如何小步迁移：把固定第三点改成函数轨迹上的动点。
- 若结构识别已稳，如何深化/抽象/包装：将面积候选点接入特殊四边形筛选。
- 禁止出的跑偏变式：只考边长计算而没有函数求点坐标条件。

## 十点五、推荐图形请求包（可选）
- 是否需要图：否。
- 图形类型：`coordinate_geometry`
- 用图意图：`student_explanation`
- 需要出现的对象：四个顶点、两条对角线、动点轨迹。
- 需要突出给学生看的关系：对角线中点相同。
- 图中不能暗示的错误性质：不要把无序构成画成唯一顺序。
- 图失败时的降级方案：使用对角点分类表。
