# 结构分析：两个函数交于一点给坐标条件

## 原题
函数求点坐标专题中的“两个函数交于一点”：题目给出两个一次函数图像相交，要求求交点坐标；或给交点的一个坐标、图像不等式分界点，反求参数并求交点。

## 一、题目场景
- 数学对象：两个一次函数 $L_1:y=f(x)$、$L_2:y=g(x)$，交点 $P$，有时含参数。
- 变量/参数：交点横坐标、纵坐标、参数 $m,b$，不等式分界点。
- 函数/图形：两条不平行的一次函数图像。
- 已知条件：两函数交于一点；可能已知交点横坐标/纵坐标，或已知 $f(x)<g(x)$ 与 $f(x)>g(x)$ 的分界。
- 要求目标：求交点坐标、求参数、由交点读不等式范围。

## 二、核心结构
### 2.1 表层信息
- 表面考点：联立一次函数、方程组、不等式图像。
- 题型功能：`basic_skill`
- 是否值得完整 structural analysis：是；本专题要锁定“交点怎样给求点坐标条件”，而不是只写联立方程。
- 一句话问题模式：两个函数交于一点，说明同一个横坐标处两个函数值相等；不等式大小关系改变的分界点就是交点横坐标。

### 2.2 结构表达
#### 判别条件表（概念辨析题用；不适用则写“无”）
- 必要条件：交点同时在两条函数图像上，具有同一个 $x$ 和同一个 $y$。
- 充分条件：点坐标代入两条函数均成立，且两函数斜率不同保证唯一交点。
- 常见干扰项：分别求两个函数上的点而不令函数值相等；忽略不等式分界处应相等。
- 最短检查动作：写 $f(x)=g(x)$。

#### 情景量表（应用题用；不适用则写“无”）
无

#### 命题网络（所有题型都写；简单题写简版）
- P1（题设）：两个一次函数 $L_1:y=f(x)$、$L_2:y=g(x)$。
- P2（定义）：$P$ 是两图像交点，所以 $P$ 同时满足 $y=f(x)$ 和 $y=g(x)$。
- P3（可推）：交点横坐标满足 $f(x)=g(x)$。
- P4（可推）：由 P3 求 $x_P$，再代回任一函数求 $y_P$。
- P5（题设/可推）：若给出 $y_P$ 或不等式分界点，可先确定交点一个坐标。
- P6（可推）：把交点坐标代入含参函数求参数，或用交点横坐标作为不等式端点。
- R1：P1 + P2 -> P3，方法：交点处两个函数值相等。
- R2：P3 -> P4，方法：解一元一次方程并代回。
- R3：P5 + P2 -> P6，方法：已知交点坐标反代参数。
- R4：P2 + 图像比较方向 -> 不等式解集，方法：以交点横坐标为分界读上下关系。
- 目标：求交点坐标、参数或不等式范围。

### 2.3 解题主链
```text
识别交点 -> 写 f(x)=g(x) 或定位分界点 -> 求交点坐标 -> 代回求参数/读不等式 -> Answer
```

### 2.4 模型标签
- model_id：linear_function_intersection_condition
- model_name：两个一次函数交点与图像不等式分界
- configuration：已知两个函数或交点部分信息，要求交点坐标、参数或比较范围。
- 可迁移方向：直接求交点、给交点一个坐标反求参数、不等式端点反推交点。
- 非同构边界：两条直线平行或重合时没有唯一交点，不能套唯一交点模型。

## 二点五、知识点/模型锚点
- 建议讲义标题：模型三：两个函数交点与相等关系
- 知识点/模型名称：一次函数交点坐标
- 核心公式/定理：交点处 $f(x)=g(x)$；不等式图像解法以交点横坐标为分界。
- 使用条件：两函数图像相交且斜率不同。
- 入口信号：“交于点 $P$”“交点横坐标为”“当 $x< a$ 时一个函数在另一个函数下方”。
- 易混边界：已知交点纵坐标时，可先在参数少的函数上定位横坐标。
- 本题如何体现：交点条件给出坐标相等关系，不等式范围给出交点分界。
- 可作为例题的结构层级：直接联立、已知一个坐标反求参数、不等式分界反求参数。

## 三、关键转化
- 最关键的转化：交点条件转化为两个函数值相等。
- 为什么降低计算量：只需解一个一元一次方程；已知一个坐标时先定位点可减少参数干扰。
- 不转化时的低效路径：分别在两条直线上求点，缺少共同点约束。

## 四、标准路径骨架
1. 先做什么：确认交点是同一个点。
2. 再做什么：写 $f(x)=g(x)$，或由题设先确定 $x_P/y_P$。
3. 建立什么关系：交点代入两条函数，必要时代入含参函数。
4. 如何求解：解一元一次方程或简单参数方程。
5. 需要检查什么：两条函数斜率不同，交点代回两式均成立，不等式端点取舍正确。

## 四点五、标准完整解与验算
- 关键交点/关键量：$y=2x+1$ 与 $y=-x+7$ 的交点。
- 面积/方程/关系式：交点处 $2x+1=-x+7$。
- 完整求解过程：解得 $3x=6$，$x=2$；代入得 $y=5$。
- 最终答案：$P(2,5)$。
- 排除值：若斜率相同则不适用唯一交点模型。
- 退化情形：两直线重合时交点不唯一。
- 验算：$2\cdot2+1=5$，$-2+7=5$。
- 本题最短可靠路径：令两个解析式相等，先求横坐标再代回。

## 五、出题人逻辑
- 诱导学生硬算的位置：被两个函数式吓到，或把不等式范围当作直接代数不等式而忽略图像分界。
- 真正的捷径：交点就是相等；不等式分界就是交点横坐标。
- 训练的可迁移能力：把交点、参数、不等式范围统一到同一个坐标条件。

## 六、学生卡点预测
- 读题/入手动作卡点：不知道交点应同时满足两条函数。
- 建模/关系入口卡点：已知纵坐标时不会先在简单函数上求横坐标。
- 求解/检查卡点：求出参数后不写交点，或不等式端点取舍错误。

## 七、变式原则
- 核心不变量：交点处两个函数值相等。
- 表层特征：两函数交于 $P$，已知交点坐标或图像大小关系。
- 可变维度：直接求交点、已知横/纵坐标、含参函数、图像不等式。
- 深化阶梯：直接联立 -> 给一个坐标反求参数 -> 不等式分界反推交点 -> 交点接入后续范围判断。
- 允许的变换：斜率、截距、参数位置、比较方向。
- 禁止的变换：两直线平行却要求唯一交点；不等式方向不明确却要求区间。
- 表征切换：解析式、方程组解、交点坐标、图像上下关系。
- 包装方式：用 $f(x),g(x)$ 表示两个函数，或用文字给出左右两侧大小关系。
- 近迁移例子：求 $y=2x+1$ 与 $y=-x+7$ 交点。
- 远迁移例子：已知大小关系在 $x=3$ 两侧改变，反求参数。
- 反例/伪变式：只给两条函数但没有交点或比较问题，只做单函数代入。

## 八、计算复杂度预算
- 原题计算层级：一个一元一次方程。
- 允许小步上升到：一个简单分数参数或一个不等式端点判断。
- 禁止引入的计算负担：复杂方程组、二次交点、多个参数同时反求。
- 必须保留的可见支架：交点、已知坐标、比较方向或分界点。

## 九、推荐讲题任务包
- 建议的本轮教学入口：用“同一个点，同一个 $x$，同一个 $y$”解释交点。
- 本题讲解目标：学生能主动写出 $f(x)=g(x)$。
- 不要直接讲的抽象话：不要只说“联立方程组”。
- 必须先问的问题：交点上的 $y$ 值由两条函数算出来，应该是什么关系？
- 关键讲解顺序：交点相等 -> 求坐标 -> 参数/不等式应用。
- 最适合的具体数值例子：$y=2x+1$ 与 $y=-x+7$。
- 讲到哪里停下来让学生回答：写出相等方程前，让学生说为什么相等。

## 十、推荐练题任务包
- 若卡在读题/入手动作，出什么题：直接求两函数交点。
- 若卡在建模或关系入口，出什么题：已知交点纵坐标，先在简单函数上求横坐标。
- 若卡在求解和检查，出什么题：不等式左右区间给出分界点，反求参数。
- 若原题已稳，如何小步迁移：把交点作为方程组解或图像不等式端点。
- 若结构识别已稳，如何深化/抽象/包装：从不等式范围反向定位交点。
- 禁止出的跑偏变式：无唯一交点却要求单点坐标。

## 十点五、推荐图形请求包（可选）
- 是否需要图：否
- 图形类型：`function_graph`
- 用图意图：`student_explanation`
- 需要出现的对象：两条相交直线。
- 需要突出给学生看的关系：交点处两个函数值相等。
- 图中不能暗示的错误性质：不要画成平行或重合。
- 图失败时的降级方案：用方程相等关系讲解。
