# 结构分析：矩形存在性综合作业

## 原题
围绕坐标系中的矩形存在性设计一组 10 题综合练习。题目覆盖固定顺序矩形、三点构成矩形、动点在直线/反比例函数/二次函数上、给定对角线、参数求值等场景。目标是让学生能稳定完成“先判定矩形结构，再列坐标关系，最后排除退化”的完整路径。

## 一、题目场景
- 数学对象：平面直角坐标系中的点、线段、四边形、函数图像。
- 变量/参数：动点坐标、直线参数、反比例函数参数、二次函数参数、矩形第四点。
- 函数/图形：一次函数、反比例函数、简单二次函数、水平/竖直线段、给定对角线。
- 已知条件：部分点坐标，动点所在轨迹，矩形顺序或“不固定顺序”的构成关系。
- 要求目标：求动点坐标、第四点坐标、参数值，或判断所有可能情形。

## 二、核心结构
### 2.1 表层信息
- 表面考点：矩形的判定、坐标距离、垂直关系、中点关系、函数图像上的点。
- 题型功能：`existence_or_parameter`
- 是否值得完整 structural analysis：是；本题组的难点不在单次计算，而在“矩形存在性”的分类入口和退化检查。
- 一句话问题模式：给定部分顶点和轨迹条件，寻找能使四边形成为矩形的点或参数。

### 2.2 结构表达
#### 判别条件表（概念辨析题用；不适用则写“无”）
- 必要条件：矩形必须先是平行四边形，且有一个角为直角；或对角线互相平分且相等。
- 充分条件：平行四边形中一角为直角；或四边形对角线互相平分且相等。
- 常见干扰项：只用平行四边形公式求第四点，忘记直角；只列一个直角位置；保留点重合的退化解。
- 最短检查动作：固定顺序看邻边点积；不固定顺序分三种直角位置或三种对角配对。

#### 情景量表（应用题用；不适用则写“无”）
| 量 | 类型/单位 | 题设关系 | 未知/已知 |
|---|---|---|---|
| A, B | 点坐标 | 通常为已知相邻点或对角端点 | 已知 |
| C | 动点 | 在直线、反比例函数、二次函数或指定直线上 | 未知 |
| D | 第四点 | 由平行四边形对角关系或邻边关系推出 | 未知 |
| 参数 | 实数 | 由直角、对角线或轨迹条件确定 | 未知 |

#### 命题网络（所有题型都写；简单题写简版）
- P1（题设）：已知若干点坐标与动点轨迹。
- P2（定义）：矩形是有一个直角的平行四边形。
- P3（构造）：固定顺序 $ABCD$ 时，$D=A+C-B$，且常用 $\overrightarrow{AB}\cdot\overrightarrow{BC}=0$。
- P4（构造）：不固定顺序时，需分别讨论 $\angle A,\angle B,\angle C$ 为直角，或分别讨论哪两个点为对角。
- P5（计算状态）：把动点写成参数点，如 $C(t,kt+b)$、$C(t,\frac{k}{t})$ 或 $C(t,f(t))$。
- P6（检查）：排除点重合、边长为 0、参数不满足范围、函数定义域不满足的情形。
- R1：P1 + P5 -> P3，方法：轨迹参数化后代入矩形直角条件。
- R2：P2 + P3 -> D，方法：平行四边形对角线中点相同，或向量和公式。
- R3：P2 + P4 -> 所有可能情形，方法：按直角顶点分类并逐一验算。
- 目标：由 P1-P6 推出所有满足矩形条件的点、参数与第四点坐标。

### 2.3 解题主链
```text
读清固定顺序/不固定顺序 -> 设动点坐标 -> 选矩形判定关系 -> 解参数 -> 求第四点 -> 检查退化和范围 -> Answer
```

### 2.4 模型标签
- model_id：coordinate_rectangle_existence
- model_name：坐标矩形存在性
- configuration：已知两到三点与轨迹条件，要求动点/第四点/参数。
- 可迁移方向：平行四边形存在性、菱形存在性、正方形存在性、函数图像上的特殊四边形。
- 非同构边界：只考矩形性质填空、纯证明题、无需存在性分类的普通坐标计算。

## 二点五、知识点/模型锚点
- 建议讲义标题：坐标系中的矩形存在性
- 知识点/模型名称：向量点积判垂直、距离平方判直角、平行四边形第四点公式、对角线中点关系。
- 核心公式/定理：$D=A+C-B$；$\overrightarrow{u}\cdot\overrightarrow{v}=0$；直角三角形距离平方关系；矩形对角线互相平分且相等。
- 使用条件：必须先确认顶点顺序或分类；动点必须满足原轨迹与范围。
- 入口信号：题干出现“若四边形 $ABCD$ 是矩形”或“若 $A,B,C,D$ 构成矩形”。
- 易混边界：固定顺序只求一种；不固定顺序一般要分类；对角线给定时更适合用“中点相同且对角线相等/直角”。
- 本题如何体现：10 题逐步从固定顺序过渡到隐藏结构和参数综合。
- 可作为例题的结构层级：存在性题的基础到中档综合。

## 三、关键转化
- 最关键的转化：把“矩形存在”转化为“平行四边形关系 + 一个直角”或“对角线条件”。
- 为什么降低计算量：不直接设四个未知点，而是用已知轨迹和矩形公式把未知量压到一个参数。
- 不转化时的低效路径：同时设 $C,D$ 四个坐标，再列四条边的平行、相等、垂直关系，方程过多且容易漏解。

## 四、标准路径骨架
1. 先做什么：判断题干是否固定顶点顺序。
2. 再做什么：把动点写成参数点。
3. 建立什么关系：固定顺序用邻边垂直；不固定顺序按直角位置分类；给定对角线时用点积或中点。
4. 如何求解：解一元一次/二次方程，代回轨迹。
5. 需要检查什么：函数定义域、范围条件、点重合、边长为 0、是否漏分类。

## 四点五、标准完整解与验算
- 关键交点/关键量：动点参数 $t$，第四点 $D$，矩形相邻边向量或对角线中点。
- 面积/方程/关系式：主要使用点积为 0、距离平方、$D=A+C-B$。
- 完整求解过程：本组练习在教师版 YAML 中逐题给出。
- 最终答案：见教师版答案速查。
- 排除值：$x\ne0$ 的反比例函数定义域；点与已知顶点重合；题干给定 $x>0$ 或参数范围。
- 退化情形：$C=A$、$C=B$、相邻边长度为 0、三点共线但未形成矩形。
- 验算：把答案代回轨迹，并检查相邻边点积为 0、第四点公式正确。
- 本题最短可靠路径：先分类，后计算，最后验算。

## 五、出题人逻辑
- 诱导学生硬算的位置：把“构成矩形”误以为只有一种顶点顺序；反复列距离公式但不分类。
- 真正的捷径：固定顺序用邻边垂直；水平边时直接看竖线；对角线题用点积或中点。
- 训练的可迁移能力：识别存在性题中的结构入口，而不是只记某个坐标公式。

## 六、学生卡点预测
- 读题/入手动作卡点：分不清“$ABCD$ 是矩形”和“$A,B,C,D$ 构成矩形”的差别。
- 建模/关系入口卡点：不会把轨迹上的动点写成 $C(t,f(t))$。
- 求解/检查卡点：漏掉退化解；答案没有代回函数；不检查正负或范围。

## 七、变式原则
- 核心不变量：矩形存在性 = 顶点关系分类 + 垂直/对角线条件 + 退化检查。
- 表层特征：点坐标、函数轨迹、参数求值、第四点求坐标。
- 可变维度：固定顺序/不固定顺序、轨迹类型、已知点角色、目标从求点变为求参数。
- 深化阶梯：固定顺序同结构换数 -> 不固定顺序分类 -> 轨迹包装 -> 对角线包装 -> 参数反求。
- 允许的变换：水平边换成斜边；一次函数换成反比例函数或简单二次函数；求 $C,D$ 换成求参数。
- 禁止的变换：同时引入复杂圆锥曲线、三角函数、根式大计算，导致主训练动作被掩盖。
- 表征切换：坐标、向量、距离平方、函数点参数化之间切换。
- 包装方式：用“点在函数图像上”“点在直线上”“某参数使其为矩形”等语言隐藏直角入口。
- 近迁移例子：已知 $A,B$ 和 $C$ 所在线，求固定顺序矩形。
- 远迁移例子：已知对角线两端和一点所在直线，求另一个顶点；已知参数函数图像上点形成矩形求参数。
- 反例/伪变式：只求平行四边形第四点但不要求直角，不属于矩形存在性核心训练。

## 八、计算复杂度预算
- 原题计算层级：以一元一次、简单一元二次、坐标加减为主。
- 允许小步上升到：一个可因式分解的一元二次；简单分数坐标。
- 禁止引入的计算负担：高次方程、复杂根式、三角函数、多个动点同时参数化。
- 必须保留的可见支架：题目前半组给出提示；后半组仍保持可手算数值。

## 九、推荐讲题任务包
- 建议的本轮教学入口：先让学生判断每题是否需要分类。
- 本题讲解目标：建立“固定顺序/不固定顺序”的第一分叉。
- 不要直接讲的抽象话：不要只说“数形结合”，要落到点积或距离平方。
- 必须先问的问题：题干有没有固定 $ABCD$ 的顺序？
- 关键讲解顺序：读顺序 -> 设动点 -> 列直角 -> 求第四点 -> 检查。
- 最适合的具体数值例子：水平边 $A(0,0),B(4,0)$，动点在 $y=x$ 上。
- 讲到哪里停下来让学生回答：列出三种直角分类后，让学生自己判断哪种有效。

## 十、推荐练题任务包
- 若卡在读题/入手动作，出什么题：固定顺序且水平边的矩形。
- 若卡在建模或关系入口，出什么题：动点在一次函数或反比例函数上的固定顺序题。
- 若卡在求解和检查，出什么题：含退化解 $C=A$ 或 $C=B$ 的题。
- 若原题已稳，如何小步迁移：从求坐标改为求参数。
- 若结构识别已稳，如何深化/抽象/包装：加入对角线给定、二次函数轨迹、无固定顺序分类。
- 禁止出的跑偏变式：计算特别重但矩形关系很浅的题。

## 十点五、推荐图形请求包（可选）
- 是否需要图：否。
- 图形类型：`coordinate_geometry`
- 用图意图：可选作讲评草图，不进入本次 PDF。
- 需要出现的对象：坐标轴、已知点、动点轨迹、矩形。
- 需要突出给学生看的关系：相邻边垂直、对角线中点相同。
- 图中不能暗示的错误性质：不要暗示所有题都是固定顺序；不要把非水平边强行画成水平。
- 图失败时的降级方案：用坐标向量和距离平方直接讲解。
