# 平行线对应边比例：A 字型与 8 字型绘图指导

## 1. 这次日志暴露的问题

| job | 日志现象 | 结论 |
|---|---|---|
| `explanation-eight-prompt.failed-before-seed` | 4 轮都在 Wolfram `RandomInstance` 阶段以 `invalid_head` 失败，没有生成预览图 | 只写“共线、平行、交于 P”仍有过多自由度，不能把母图交给随机求解 |
| `practice-parallel-student-b3-prompt.failed-flat-20260712-152653` | 确定性 audit 通过，但人工视觉检查认为图近乎共线、不是清楚的 8 字型 | 数学门禁只能保证关系成立，不能保证交叉角、上下间距和版面可读性 |
| `practice-parallel-student-b3-prompt` | 最后一轮只得到 A、B，C/D/P 缺失，marker 编译失败 | 修复时必须检查真正写入该 slot 的 `request.json`；不能只改相似的另一个题图 slot |
| `explanation-eight-prompt` | 加入明确坐标种子后第 1 轮即通过 | 这类母图应使用确定性坐标，不应随机采样 |

核心经验：**先固定一个不会退化的母图，再让题目的比例数据只存在于题干中。** 同一批 A 字型或 8 字型练习可以复用母图；没有必要为每道只换数字的题重新随机生成几何。

## 2. 通用绘图契约

- 原题图只画 `A,B,C,D,P`、四条题设线段和 `AB\parallel CD` 标记。
- 原题图不画相似三角形名称、对应边箭头、份数、比例推导或答案。
- 点序必须同时写进 `source_problem_text` 和 `semantic_constraints`，并在预览中人工复核。
- 坐标只作为内部视觉种子，不显示给学生。
- 不用“随机找一组满足平行和共线的点”；直接给出五点坐标或一个确定性参数公式。
- 先看最终 PNG，再看 gate。gate 通过但图扁平、交叉角过小或标签挤在一起，仍应判失败。

## 3. A 字型的稳定母图

### 3.1 推荐坐标

使用对称的两条射线：

```text
P = (0, 0)
A = (-1, 2)    B = (1, 2)
C = (-2.5, 5)  D = (2.5, 5)
```

这样自动满足：

```text
P-A-C
P-B-D
AB ∥ CD
```

通用参数式为：

```text
P = (0, 0)
A = (-u, h)       B = (u, h)
C = (-k u, k h)   D = (k u, k h)
```

建议取 `u=1`、`h=2`、`2≤k≤3`。`k` 太接近 1 时两条平行线挤在一起；`h/u` 太大时图会变成过窄的尖三角形。

### 3.2 必须画的线

- 截线：`PC`、`PD`，或分段画成 `PA+AC`、`PB+BD`。
- 平行边：`AB`、`CD`。
- 平行标记只标在 `AB`、`CD` 上。

### 3.3 视觉门禁

- `P` 到 `AB`、`CD` 的垂直距离要明显不同，不能看成同一条线。
- 两条射线夹角建议在 `30°～80°`。
- `A、B` 与 `C、D` 的标签放在图形外侧；`P` 放在顶点外侧。
- 避免把 A、B、C、D 全放在 P 的同一侧同一象限。日志中的 `a1` 虽然数学成立，但斜且窄，不适合作为默认母图。

### 3.4 可直接放入请求的文字

```yaml
source_problem_text: >-
  画清楚的 A 字型。采用内部视觉种子 P=(0,0)、A=(-1,2)、B=(1,2)、
  C=(-2.5,5)、D=(2.5,5)。点序严格为 P-A-C、P-B-D，连接 PC、PD、
  AB、CD，并标出 AB 平行 CD。坐标不显示给学生；不要画相似三角形、
  对应边箭头、比例推导或答案。
semantic_constraints:
  given_constraints:
    - P, A, C are collinear with exact order P-A-C
    - P, B, D are collinear with exact order P-B-D
    - AB is parallel to CD
    - A and B lie strictly between P and C,D respectively
    - use the fixed nondegenerate visual seed
```

## 4. 8 字型的稳定母图

### 4.1 推荐坐标

最稳妥的是上下两条水平线、交点居中：

```text
A = (-3, 2)   B = (3, 2)
P = (0, 0)
D = (-3,-2)   C = (3,-2)
```

连接 `A-P-C` 与 `B-P-D`，即可得到严格点序；`AB` 与 `CD` 自动平行。

若希望上下两个三角形大小不同，可使用：

```text
A = (-u, h)       B = (u, h)
P = (0, 0)
D = (-k u,-k h)   C = (k u,-k h)
```

建议 `u=3`、`h=2`、`0.8≤k≤1.5`。这是确定性相似缩放，始终保证两条对角线穿过 P。

### 4.2 为什么不能只写关系

“`A-P-C`、`B-P-D`、`AB∥CD`、P 在两平行线之间”只限定了拓扑关系，没有限定：

- 两条平行线相隔多远；
- 两条截线的交叉角多大；
- P 离端点是否太近；
- 图形的长宽比。

因此可能出现 audit 通过但视觉近乎一条直线的情况。坐标母图必须同时锁定这些版面自由度。

### 4.3 视觉门禁

- 上下平行线间距至少占图形总高度的 `70%`。
- 两条截线的锐角建议不小于 `35°`。
- P 到 A、B、C、D 的距离均不小于对应截线总长的 `25%`。
- `AB`、`CD` 的长度不要超过上下间距的 3 倍，防止图过扁。
- A、B 标签放上方，C、D 放下方，P 放在交点旁且不压线。

### 4.4 可直接放入请求的文字

```yaml
source_problem_text: >-
  画清楚、非退化的交叉 8 字型。采用内部视觉种子 A=(-3,2)、B=(3,2)、
  C=(3,-2)、D=(-3,-2)、P=(0,0)。画 AB、CD、AC、BD；点序严格为
  A-P-C、B-P-D，AB 平行 CD。上下平行线要有明显间距，交叉角不能过小。
  坐标不显示给学生；不要画相似三角形、对应边箭头、比例推导或答案。
semantic_constraints:
  given_constraints:
    - A, P, C are collinear with exact order A-P-C
    - B, P, D are collinear with exact order B-P-D
    - AB is parallel to CD
    - P lies inside both AC and BD
    - use the fixed nondegenerate visual seed
    - the crossing angle is at least 35 degrees visually
```

## 5. 多道题的复用策略

- A 字型三道题可共用一幅干净 A 字母图；8 字型三道题可共用一幅干净 8 字母图。
- 题目的 `PA:AC`、`PB:BD` 等数值只写在 `stem_latex`，不要做成图上的长度标记。
- 若要让版面略有变化，只允许整体镜像、上下翻转或等比例缩放；不要重新随机求点。
- 教师版没有新增几何对象时，直接复用学生版 prompt slot，不再生成一次。

## 6. 运行前后的检查

1. 在 plan YAML 中写入坐标母图。
2. collect 后检查真正生成的请求，而不是只检查 plan：

   ```bash
   jq '{source_problem_text:.problem_context.source_problem_text,
        constraints:.semantic_constraints.given_constraints}' \
     build/diagram/jobs/<job-id>/request.json
   ```

3. 如果失败 job 已经留下旧 `request.json`，先把该 job 目录改名保留，再 collect；不要覆盖失败证据。
4. 只重跑失败 job，成功 job 不重做。
5. gate 通过后查看最终选中 round 的 PNG，人工检查点序、交叉角、上下间距和标签。

## 7. 最终验收清单

- [ ] A 字型严格显示 `P-A-C`、`P-B-D`。
- [ ] 8 字型严格显示 `A-P-C`、`B-P-D`。
- [ ] `AB∥CD` 有且只有必要的平行标记。
- [ ] 没有近乎共线、过扁、过窄或 P 贴近端点的情况。
- [ ] 标签不压在线段、交点或平行标记上。
- [ ] prompt 图没有答案泄露。
- [ ] 最终 PNG 人工检查通过，而不只是 JSON audit 通过。
