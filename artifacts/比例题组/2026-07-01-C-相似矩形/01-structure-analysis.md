# 结构分析：相似矩形

## 原题
围绕“判断对应关系 -> 建比例 -> 解比例”设计相似矩形训练：

1. 矩形甲长 $4$，宽 $6$；矩形乙长 $8$，宽 $12$。判断两个矩形是否相似。
2. 两个矩形相似。矩形甲长 $4$，宽 $6$；矩形乙长 $8$，宽 $x$。求 $x$。
3. 一个矩形两边长为 $2$ 和 $3$，另一个矩形一边长为 $6$，另一边长为 $x$。若两个矩形相似，求 $x$。
4. 两个矩形相似。矩形甲长 $x+1$，宽 $8$；矩形乙长 $3$，宽 $4$。求 $x$。

## 一、题目场景
- 数学对象：矩形、长边、短边、相似比例。
- 变量/参数：未知边 $x$。
- 函数/图形：无。
- 已知条件：两个矩形相似或给出两组边长。
- 要求目标：判断是否相似或求未知边长。

## 二、核心结构
### 2.1 表层信息
- 表面考点：相似矩形。
- 题型功能：`application_modeling`。
- 是否值得完整 structural analysis：是；核心不是比例计算本身，而是先判断对应边。
- 一句话问题模式：先比较长宽比或确定对应边，再列比例求未知量。

### 2.2 结构表达
#### 判别条件表（概念辨析题用；不适用则写“无”）
- 必要条件：两个矩形长宽比相同。
- 充分条件：长边对应长边、短边对应短边后比例相等。
- 常见干扰项：图形旋转后对应关系混乱；没有说明 $6$ 对应哪条边时漏掉两种情况。
- 最短检查动作：先把每个矩形的两边按长短排序。

#### 情景量表（应用题用；不适用则写“无”）
| 量 | 类型/单位 | 题设关系 | 未知/已知 |
|---|---|---|---|
| 矩形甲长宽 | 长度 | 形成长宽比 | 已知或含未知 |
| 矩形乙长宽 | 长度 | 与甲相似时对应成比例 | 已知或含未知 |
| $x$ | 长度 | 由比例方程求出 | 未知 |

#### 命题网络（所有题型都写；简单题写简版）
- P1（题设）：给出两个矩形的边长。
- P2（定义）：矩形相似要求长宽比相同。
- P3（可推）：对应边关系。
- P4（可推）：比例方程。
- P5（目标）：未知边或相似判断。
- R1：P1 + P2 -> P3，方法：长边对长边、短边对短边。
- R2：P3 -> P4，方法：列对应边比例。
- R3：P4 -> P5，方法：倍数法、份数法或交叉相乘。
- 目标：判断相似或求边长。

### 2.3 解题主链
```text
边长排序 -> 判断对应边 -> 写比例式 -> 解比例 -> 检查是否有多对应
```

### 2.4 模型标签
- model_id：similar_rectangle_correspondence
- model_name：相似矩形对应边比例
- configuration：已知矩形边长或相似关系，求判断或未知边。
- 可迁移方向：一般相似图形、图形旋转后的对应关系。
- 非同构边界：未给相似且长宽比不同，不能硬列比例求边。

## 二点五、知识点/模型锚点
- 建议讲义标题：相似矩形：先找对应边
- 知识点/模型名称：长宽比、对应边、比例方程、多对应讨论。
- 核心公式/定理：相似矩形的对应边成比例。
- 使用条件：两个对象都是矩形；对应边判断清楚。
- 入口信号：“两个矩形相似”“判断是否相似”“另一边为 $x$”。
- 易混边界：没有说明哪条边对应时，可能有两种答案。
- 本题如何体现：从判断相似到多解讨论，再到含代数式比例。
- 可作为例题的结构层级：比例工具在相似场景中的应用。

## 三、关键转化
- 最关键的转化：先确定对应边，而不是直接把题面顺序当作对应顺序。
- 为什么降低计算量：对应关系正确后，比例方程很短。
- 不转化时的低效路径：盲目用题面顺序列式，导致长边对短边。

## 四、标准路径骨架
1. 先做什么：写出每个矩形的长宽比。
2. 再做什么：判断是否相似或确定对应边。
3. 建立什么关系：对应边比例。
4. 如何求解：比例方程求 $x$。
5. 需要检查什么：是否存在两种对应情况。

## 四点五、标准完整解与验算
- 关键交点/关键量：对应边。
- 面积/方程/关系式：
  - $4:6=8:12=2:3$，相似。
  - $4:6=8:x$，$x=12$。
  - 若 $6$ 对应 $2$，$x=9$；若 $6$ 对应 $3$，$x=4$。
  - $\dfrac{x+1}{8}=\dfrac{3}{4}$，$x=5$。
- 完整求解过程：先判对应边，再列比例。
- 最终答案：相似；$x=12$；$x=9$ 或 $4$；$x=5$。
- 排除值：边长为正。
- 退化情形：边长为 $0$ 不构成矩形。
- 验算：回代后长宽比一致。
- 本题最短可靠路径：边长排序后列比例。

## 五、出题人逻辑
- 诱导学生硬算的位置：直接按题面顺序列比例。
- 真正的捷径：先确定对应关系。
- 训练的可迁移能力：相似场景下的比例建模。

## 六、学生卡点预测
- 读题/入手动作卡点：不知道先比较长宽比。
- 建模/关系入口卡点：旋转或未指定对应时漏情况。
- 求解/检查卡点：算出一个 $x$ 后不回代检查。

## 七、变式原则
- 核心不变量：对应边成比例。
- 表层特征：数字、未知边、边长给出顺序。
- 可变维度：判断相似、求边、讨论多解、含一次式。
- 深化阶梯：判断 -> 倍数求边 -> 多对应 -> 交叉相乘。
- 允许的变换：简单整数和分数边长。
- 禁止的变换：一般四边形无相似条件。
- 表征切换：边长列表、文字描述、旋转矩形。
- 包装方式：不给长宽名称，只说两边长。
- 近迁移例子：相似三角形对应边。
- 远迁移例子：面积比中先找对应底或高。
- 反例/伪变式：两个矩形边长 $4,6$ 与 $8,10$，不能说相似。

## 八、计算复杂度预算
- 原题计算层级：一层比例。
- 允许小步上升到：一个一次式。
- 禁止引入的计算负担：复杂代数式或二次方程。
- 必须保留的可见支架：长宽比、对应边、比例式。

## 九、推荐讲题任务包
- 建议的本轮教学入口：先问“谁对应谁”。
- 本题讲解目标：不把题面顺序误当对应顺序。
- 不要直接讲的抽象话：不要只说“相似比”。
- 必须先问的问题：长边是哪条？短边是哪条？
- 关键讲解顺序：判断相似 -> 求边 -> 多解 -> 代数式比例。
- 最适合的具体数值例子：$2,3$ 与 $6,x$。
- 讲到哪里停下来让学生回答：列出两种对应情况后停。

## 十、推荐练题任务包
- 若卡在读题/入手动作，出什么题：判断两个矩形是否相似。
- 若卡在建模或关系入口，出什么题：旋转后求对应边。
- 若卡在求解和检查，出什么题：含未知边比例方程。
- 若原题已稳，如何小步迁移：不给对应边，要求分类。
- 若结构识别已稳，如何深化/抽象/包装：含一次式边长。
- 禁止出的跑偏变式：没有相似条件的任意四边形。

## 十点五、推荐图形请求包（可选）
- 是否需要图：是。
- 图形类型：`synthetic_geometry`
- 用图意图：学生讲解与练习题题面均配矩形示意，帮助学生先判断对应关系。
- 需要出现的对象：两组矩形，顶点标签与题干边长描述对应。
- 需要突出给学生看的关系：长边、短边、以及未指定对应时可能有两种对应。
- 图中不能暗示的错误性质：不要暗示题目未给出的唯一对应关系。
- 图失败时的降级方案：不得直接编译无图版本，应回到 diagram workflow 修复图位。
